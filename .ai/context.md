# AI Project Context

Version: 1.0.0

---

# Project Name

Platform Engineering & AI Infrastructure Course

---

# Mission

Build the highest-quality free, open-source Platform Engineering & AI Infrastructure curriculum.

The curriculum should rival or exceed commercial learning platforms by emphasizing:

* Technical accuracy
* Hands-on learning
* Production-ready engineering
* Real-world projects
* Career-focused outcomes
* Continuous improvement

The goal is to develop capable engineers—not simply learners who memorize commands.

---

# Project Objectives

The curriculum should enable learners to:

* Learn modern infrastructure technologies.
* Build production-quality projects.
* Think like engineers.
* Troubleshoot complex systems.
* Build portfolio-ready work.
* Prepare for engineering interviews.
* Progress from beginner to professional.

# AI Responsibilities

Every AI agent should:

* Read the required project documents.
* Follow templates.
* Produce modular output.
* Explain assumptions.
* Prioritize correctness.
* Recommend improvements separately from approved content.

---

# AI Limitations

Agents must never:

* Invent technical facts.
* Ignore project standards.
* Skip required lesson sections.
* Generate incomplete deliverables.
* Replace official documentation with unsupported opinions.

---

# Engineering Principles

Always teach:

* Why something exists.
* How it works.
* Trade-offs.
* Failure scenarios.
* Troubleshooting.
* Production considerations.
* Security implications.
* Scalability.
* Maintainability.

The objective is to teach engineering judgment, not memorization.

---

# Quality Expectations

Every generated artifact should be:

* Technically correct.
* Educationally valuable.
* Production-aware.
* Version-aware.
* Consistent.
* Maintainable.

Anything that does not satisfy these expectations should be revised before approval.
