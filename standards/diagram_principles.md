# Diagram Principles

These principles must be strictly followed when creating architectural, infrastructural, or conceptual diagrams for TechFliq Learn.

- **Never sacrifice correctness for aesthetics.**
- **Every arrow must represent a real interaction.**
- **Don't imply direct communication where intermediaries exist.**
- **Simplify by omitting detail, not by inventing connections.**
- **Use layered architecture where appropriate.**
- **Label boundaries like User Space and Kernel Space.**
- **Use consistent colors across all diagrams (e.g., blue for user space, green for kernel, orange for hardware).**
- **Prefer SVG or high-quality vector diagrams for core concepts.**
