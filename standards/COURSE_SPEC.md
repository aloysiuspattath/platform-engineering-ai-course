# COURSE SPECIFICATION

## 1. Project Overview

**Project Name**
Open Platform Engineering & AI Infrastructure Curriculum

**Version**
0.1.0

**Status**
Planning

---

## 2. Vision

Create the highest-quality free, open-source Platform Engineering and AI Infrastructure course that rivals or exceeds commercial learning platforms.

The course should prepare learners for real engineering roles by emphasizing practical skills, production-ready projects, and modern infrastructure practices.

---

## 3. Target Audience

Primary:

* Students
* Developers
* Linux Administrators
* DevOps Engineers
* Cloud Engineers
* System Administrators

Secondary:

* Platform Engineers
* SREs
* AI Engineers
* MLOps Engineers

---

## 4. Prerequisites

No prior Platform Engineering knowledge required.

The course should teach required fundamentals before advanced topics.

---

## 5. Learning Path

Each module should build upon previous modules.

Never introduce advanced topics before foundational concepts.

---

## 6. Every Module Must Include

* Learning objectives
* Prerequisites
* Theory
* Visual explanations
* Practical demonstrations
* Hands-on lab
* Mini project
* Summary
* Cheat sheet
* Quiz
* Interview questions
* Further reading

---

## 7. Every Lab Must Include

* Goal
* Estimated time
* Difficulty
* Prerequisites
* Environment setup
* Step-by-step instructions
* Verification
* Troubleshooting
* Cleanup

---

## 8. Every Project Must Include

* Business scenario
* Architecture diagram
* Folder structure
* Requirements
* Implementation
* Testing
* Deployment
* Monitoring
* Suggested improvements

---

## 9. Preferred Technologies

Prioritize modern, widely adopted, open-source technologies where practical.

Examples include Linux, Git, Docker, Kubernetes, Terraform, Prometheus, Grafana, OpenTelemetry, PostgreSQL, Python, Go, Ollama, and vLLM.

Commercial tools may be introduced later for comparison.

---

## 10. Documentation Standards

Use Markdown.

Keep lessons modular.

Include diagrams where helpful.

Prefer code examples over long theoretical explanations.

---

## 11. Quality Standards

Content must be:

* Technically accurate
* Beginner friendly
* Production relevant
* Version aware
* Well structured
* Regularly reviewable

---

## 12. Final Outcome

By completing the curriculum, learners should be capable of designing, deploying, operating, and troubleshooting modern platform engineering and AI infrastructure systems with confidence.
